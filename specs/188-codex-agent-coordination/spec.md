# Feature Specification: Configure Codex to skip tickets claimed by Claude Code

**Feature Branch**: `188-codex-agent-coordination`
**Created**: 2026-04-07
**Status**: Approved (retroactively, 2026-07-18 — see decision-log.md Decision 26; implementation independently verified complete in AGENTS.md and docs/multi-thread-workflow.md)
**Input**: GitHub issue #188

> **Governance note**: Dev tooling only. No runtime capability, no contract changes. Codex is human-triggered via starter prompts — the coordination mechanism lives in instructions and AGENTS.md, not in automated queue scanning.

## Context

Codex is invoked manually with a per-issue starter prompt (see `docs/multi-thread-workflow.md`). It has no automated ticket scanner. Claude Code uses the `claim-ticket` skill (introduced in #183) to label issues `agent:claude` before starting work. Codex needs equivalent awareness so a human starting a Codex thread doesn't accidentally pick up an already-claimed issue.

## User Scenarios & Testing

### User Story 1 - Codex starter prompt warns on claimed ticket (Priority: P1)

As a developer starting a Codex dev thread, I want the starter prompt to instruct Codex to check for `agent:claude` label and existing `claude/issue-NNN-*` branches before beginning, so that Codex stops and reports if the ticket is already claimed by Claude Code.

**Why this priority**: This is the primary guard. All other stories depend on this check being present in the thread context.

**Independent Test**: Start a Codex thread on an issue labeled `agent:claude`. Codex should output a conflict message and halt without creating a branch or PR.

**Acceptance Scenarios**:

1. **Given** issue #N has label `agent:claude`, **When** Codex reads the starter prompt, **Then** Codex stops and reports the issue is claimed by Claude Code.
2. **Given** a `claude/issue-N-*` branch exists remotely, **When** Codex checks before starting, **Then** Codex stops and reports a branch conflict.
3. **Given** issue #N has no agent label and no claude branch, **When** Codex starts, **Then** Codex proceeds and claims with `agent:codex`.

---

### User Story 2 - Codex claims ticket on start (Priority: P2)

As a developer, I want Codex to claim a ticket by setting `agent:codex` label and Agent → Codex on Project 1 when it begins work, so that Claude Code's pre-flight check (in the `claim-ticket` skill) can detect the conflict.

**Why this priority**: Symmetric to Claude Code's claim-ticket skill. Without this, Claude Code won't know Codex is working on an issue.

**Independent Test**: Start a Codex thread on a free issue. Verify `agent:codex` label and Agent → Codex field are set before any code changes.

**Acceptance Scenarios**:

1. **Given** a free issue, **When** Codex starts, **Then** `agent:codex` label is added and Agent → Codex is set on Project 1.
2. **Given** Codex has claimed an issue, **When** Claude Code runs `claim-ticket` on the same issue, **Then** Claude Code's pre-flight check detects `agent:codex` and stops.

---

### Edge Cases

- What if the label check fails (GitHub API error)? Codex should stop and ask the developer to verify manually rather than proceeding blindly.
- What if the issue has both `agent:claude` and `agent:codex`? Treat as claimed — stop and report.

## Requirements

- **FR-001**: The Codex dev thread starter prompt MUST include a pre-flight check for `agent:claude` label and `claude/issue-NNN-*` branches before starting work.
- **FR-002**: The Codex dev thread starter prompt MUST instruct Codex to claim the ticket by adding `agent:codex` label and setting Agent → Codex on Project 1 when the pre-flight passes.
- **FR-003**: `AGENTS.md` MUST include the coordination rules so they are present in Codex's session context regardless of how it is started.
- **FR-004**: `docs/multi-thread-workflow.md` MUST be updated to document the two-agent coordination model.

## Success Criteria

- **SC-001**: A Codex thread started on an `agent:claude`-labeled issue stops before creating a branch.
- **SC-002**: A Codex thread started on a free issue adds `agent:codex` label before any code change.
- **SC-003**: Claude Code's `claim-ticket` skill detects `agent:codex` and stops if Codex has claimed the issue.
- **SC-004**: `cargo test` passes unchanged — no Rust code is touched.

## Assumptions

- Codex is always started manually with the dev thread starter prompt.
- Codex can run `gh` CLI commands to check labels and branches.
- AGENTS.md is auto-generated by `update-agent-context.sh` but preserves manual additions between markers.
